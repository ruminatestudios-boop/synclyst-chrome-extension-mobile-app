Connect the extension to localhost

The extension talks to Next (this app). Photo extraction uses the Python API on port 8000 on your machine — even when you open /snap from your phone, Next on your Mac calls localhost:8000. You must run the backend and Next together (see below).

Do this once

  1. From the repo root, start backend + Next in one terminal (needs auralink-ai/backend/.venv with deps installed): 
    npm run dev:synclyst
    Wait until you see both the API and Next ready. This page should load on http://127.0.0.1:3000/local-test (or :3001 if 3000 is busy).
  2. Generate the Chrome dev manifest (localhost + LAN): 
    npm run extension:manifest:dev
  3. Chrome → chrome://extensions → Load unpacked → choose the extension folder → Reload after each manifest change.
  4. If you ever pointed the extension at production, reset storage (popup → right‑click → Inspect → Console): 
    chrome.storage.local.remove(["synclyst_origin","synclyst_origin_auto"])
    Then reload the extension.
